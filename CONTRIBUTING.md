# Contribution Guidelines

Thank you for your interest in contributing to SENTIENCE! Please follow these guidelines to help us maintain a high quality of work in the project.

## How to Contribute
1. **Fork the Repository:** Click on the fork button to create a copy of the repo in your own GitHub account.
2. **Create a Branch:** Create a new branch for your feature or bug fix.
3. **Make Changes:** Implement your changes and ensure your code adheres to our coding standards.
4. **Testing:** Write tests for your code and confirm that all tests pass.
5. **Commit Changes:** Commit your changes with a clear message describing what you have done.
6. **Push to GitHub:** Push your branch back to your GitHub account.
7. **Open a Pull Request:** Go to the original repository and open a pull request, explaining your changes in detail.

## Code of Conduct
Contributors are expected to adhere to the [code of conduct](CODE_OF_CONDUCT.md).

## Questions
If you have any questions, feel free to open an issue or reach out to us directly.

Happy contributing!
